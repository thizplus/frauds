# ระบบเก็บข้อมูลลูกหนี้ (Lender CRM) — แผนการทำ

## ภาพรวม

ระบบสำหรับ **เจ้าของเงิน (ผู้ปล่อยกู้/เจ้าของวงแชร์/นายทุน)** ให้สามารถ:
1. เก็บข้อมูลลูกหนี้/ลูกค้าทั้งหมดไว้ที่เดียว
2. ส่งลิงก์ให้ลูกหนี้กรอกข้อมูลตัวเอง (ไม่ต้อง login)
3. ค้นหาลูกหนี้ในระบบ → กด "แจ้งโกง" ได้ทันที (verified=true ไม่ต้อง admin approve)
4. ปลดโกงได้เมื่อชดใช้แล้ว

---

## User Flow

### Flow 1: เจ้าของเงินเปิดใช้ระบบ
```
เจ้าของเงิน (login แล้ว)
  → กด "เปิดระบบเก็บข้อมูล" (ครั้งเดียว)
  → ระบบสร้าง invite link ให้ (unique URL)
  → เจ้าของ copy ลิงก์ → ส่งให้ลูกหนี้/ลูกค้าทุกคน
  → ลูกหนี้ทุกคนกรอกข้อมูลผ่านลิงก์เดียวกัน → เข้ามารวมอยู่ในระบบ
```

### Flow 2: ลูกหนี้ลงทะเบียน
```
ลูกหนี้ได้รับลิงก์ → เปิดลิงก์
  → หน้าลงทะเบียน (ไม่ต้อง login)
  → เห็นแค่: "ลงทะเบียนข้อมูล" — ไม่มีคำว่า "โกง" หรือ "ตรวจสอบ"
  → กรอก: ชื่อ, นามสกุล, เลข ปชช, เบอร์โทร, เลขบัญชี, ธนาคาร, ที่อยู่
  → ถ่ายรูปบัตร ปชช / selfie (ยืนยันตัวตน)
  → ส่ง → "ลงทะเบียนสำเร็จ"
  → *** ลูกหนี้ไม่รู้ว่าข้อมูลจะถูกเช็คกับระบบ ***
  → *** สมัครกี่ที่ก็ได้ ไม่มีการบอกว่าซ้ำ ***
```

### Flow 3: เช็คประวัติลูกหนี้
```
เจ้าของเงินกดปุ่ม "เช็คประวัติ" ที่ลูกหนี้คนไหนก็ได้
  → ระบบค้นหาจาก: เบอร์ / บัญชี / บัตร ปชช / ชื่อ-นามสกุล
  → แสดงผลลัพธ์ตามที่พบ (ไม่ตัดสิน ไม่ให้ label):

  แสดง: จำนวน records ที่ match + รายละเอียดแต่ละ record
  - แหล่งข้อมูล (fraud report / bot collector / social)
  - จำนวนครั้งที่ถูกแจ้ง
  - วันที่
  - field ที่ match (เบอร์/บัญชี/ชื่อ)

  → เจ้าของเงินดูข้อมูลแล้วตัดสินใจเอง
  → กดปุ่ม "เช็คอีกครั้ง" ได้ (กรณีข้อมูลอัปเดต)

  * ระบบแค่แสดงข้อมูลที่พบ ไม่ระบุว่า safe/danger
  * รองรับ data source หลายแหล่ง (frauds, bot collector, social)
```

### Flow 4: เจ้าของเงินจัดการ
```
เจ้าของเงินเข้าหน้าจัดการ
  → เห็นรายชื่อลูกหนี้ทั้งหมด + สถานะเช็คประวัติ
  → ค้นหาด้วย ชื่อ / เบอร์ / เลข ปชช
  → กดดูรายละเอียด

  เมื่อลูกหนี้มีปัญหา (ไม่จ่าย/หนีหนี้):
  → กด "แจ้งโกง"
  → Dialog: กรอกเหตุผล + จำนวนเงิน + รายละเอียด
  → ยืนยัน → ข้อมูลเข้า frauds ทันที (verified=true)
  → ลูกหนี้ถูก mark สีแดง

  เมื่อลูกหนี้ชดใช้แล้ว:
  → กด "ปลดโกง"
  → Dialog: กรอกหมายเหตุ
  → ยืนยัน → ลบออกจากผลค้นหา → ลูกหนี้กลับเป็นปกติ
```

## Data Model

### ตารางใหม่

```
lender_profiles (ข้อมูลเจ้าของเงิน — 1 user = 1 profile)
├── id               UUID PK
├── user_id          UUID FK → users (UNIQUE — 1 คนมีได้ 1 ระบบ)
├── business_name    "ร้านกู้เงินสมชาย", "วงแชร์บ้านนาย"
├── type             "loan" | "share" | "trade" | "other"
├── invite_code      VARCHAR(20) UNIQUE — สำหรับ invite URL
├── is_active        BOOLEAN
├── created_at
├── updated_at

debtors (ลูกหนี้/ลูกค้า — เก็บรวมทั้งหมดไม่แยกวง)
├── id               UUID PK
├── lender_id        UUID FK → lender_profiles
├── first_name
├── last_name
├── id_card          เลข ปชช
├── phone
├── bank_account
├── bank_name
├── address          ที่อยู่
├── social_accounts  JSONB (LINE, FB, IG)
├── id_card_image    URL รูปบัตร ปชช (R2)
├── selfie_image     URL รูป selfie (R2)
├── note             หมายเหตุเจ้าของเงิน
├── check_matches    INT — จำนวน records ที่ match (0 = ยังไม่เช็ค หรือ ไม่พบ)
├── check_result     JSONB — ผลเช็คล่าสุด (array of matched records)
├── checked_at       เวลาที่เช็คล่าสุด (null = ยังไม่เคยเช็ค)
├── status           "active" | "flagged" | "cleared"
├── fraud_id         UUID FK → frauds (null ถ้ายังไม่แจ้ง)
├── flagged_at       เวลาที่แจ้งโกง
├── flagged_reason   เหตุผล
├── flagged_amount   จำนวนเงินที่โกง (satang)
├── flagged_detail   รายละเอียดเพิ่มเติม
├── cleared_at       เวลาที่ปลดโกง
├── cleared_note     หมายเหตุปลดโกง
├── created_at
├── updated_at
```

### ความสัมพันธ์

```
ลูกหนี้ 1 คน สมัครกับหลายเจ้ามือ:

users (เจ้าของเงิน)
  └── lender_profiles (1:1)
       └── debtors (1:N — ลูกหนี้ของเจ้ามือนี้)

ลูกหนี้คนเดียวกัน (เบอร์/บัตร ปชช เดียวกัน) สมัครกับ 3 เจ้ามือ:

  เจ้ามือ A → debtor record #1 (สมชาย, 081-xxx)
  เจ้ามือ B → debtor record #2 (สมชาย, 081-xxx)
  เจ้ามือ C → debtor record #3 (สมชาย, 081-xxx)

  = 3 records แยกกัน ไม่ merge
  = แต่ละเจ้ามือเห็นแค่ของตัวเอง
  = ตอนเช็คประวัติ → ระบบค้นหาข้ามทุก source รวมถึง
    debtor ที่ถูก flag โดยเจ้ามือคนอื่น
```

### หลักการออกแบบ

| หลักการ | รายละเอียด |
|---------|-----------|
| **ลูกหนี้ไม่รู้** | ไม่แสดงว่ามีข้อมูลในระบบ, หน้าลงทะเบียนเหมือนกรอกข้อมูลธรรมดา |
| **ไม่ merge ข้าม** | debtor ของเจ้ามือ A ≠ debtor ของเจ้ามือ B แม้เป็นคนเดียวกัน |
| **เจ้ามือเห็นแค่ของตัวเอง** | query by lender_id เสมอ |
| **เช็คประวัติข้ามได้** | ตอนกดเช็ค → ค้นหาจาก frauds + debtor ที่ถูก flag ทั้งระบบ |
| **Flag ไม่กระทบข้าม** | เจ้ามือ A แจ้งโกง → เข้า frauds → เจ้ามือ B เช็คเจอ (ผ่านผลเช็ค) |

## API Endpoints

### Lender Profile (JWT auth)
```
POST   /lender/setup              เปิดใช้ระบบ (สร้าง profile + invite code)
GET    /lender/profile             ดูข้อมูลระบบของฉัน
PUT    /lender/profile             แก้ไขข้อมูล
GET    /lender/invite-link         ดู invite link
```

### Debtor Management (JWT auth — เจ้าของเงินเท่านั้น)
```
GET    /lender/debtors                     รายชื่อลูกหนี้ทั้งหมด (search + pagination)
GET    /lender/debtors/:id                 ดูรายละเอียดลูกหนี้
POST   /lender/debtors                     เพิ่มลูกหนี้ด้วยมือ
PUT    /lender/debtors/:id                 แก้ไขข้อมูลลูกหนี้
DELETE /lender/debtors/:id                 ลบลูกหนี้
POST   /lender/debtors/:id/check            เช็คประวัติกับ frauds
POST   /lender/debtors/:id/flag            แจ้งโกง
POST   /lender/debtors/:id/clear           ปลดโกง
```

### Public Registration (ไม่ต้อง login)
```
GET    /register/:code                     ดูข้อมูลเจ้าของ (ชื่อร้าน)
POST   /register/:code                     ลูกหนี้ลงทะเบียน
```

## Backend Flows

### เช็คประวัติ (Manual — เจ้าของกดเอง)
```
POST /lender/debtors/:id/check

Backend:
1. ดึง debtor data (ชื่อ, เบอร์, บัญชี, บัตร ปชช)
2. Query จากทุก data source:
   a) frauds — บัตร ปชช / เบอร์ / บัญชี / ชื่อ-นามสกุล
   b) debtors ที่ถูก flag โดยเจ้ามือคนอื่น — เบอร์ / บัญชี / บัตร ปชช
   c) อนาคต: bot_collected_posts, social_reports
3. รวม matched records เป็น array:
   [
     { source: "fraud_report", matchedBy: "phone", reportCount: 3, ... },
     { source: "lender_flag", matchedBy: "id_card", flaggedBy: "เจ้ามือ X", ... },
     { source: "bot_collector", matchedBy: "bank_account", ... },
   ]
4. *** ไม่เปิดเผยว่าเจ้ามือคนไหน flag ***
   → แสดงแค่ "ถูกแจ้งโดยผู้ใช้ในระบบ" ไม่ระบุชื่อ
5. อัปเดต debtor: check_matches, check_result (JSONB), checked_at
6. Return: matched records array (ไม่ตัดสิน ไม่ให้ label)

* ระบบแค่ค้นหาแล้วแสดงผล — เจ้าของเงินตัดสินใจเอง
* ข้อมูลข้ามเจ้ามือ: เห็นว่ามีคนอื่นแจ้ง แต่ไม่รู้ว่าใคร
* ออกแบบให้เพิ่ม data source ได้ง่าย (Port/Adapter pattern)
```

### ตัวอย่าง: สมชาย สมัคร 3 ที่
```
เจ้ามือ A: สมชาย ลงทะเบียน → ยังไม่มีปัญหา → สถานะ: ปกติ
เจ้ามือ B: สมชาย ลงทะเบียน → ไม่จ่ายเงิน → เจ้ามือ B กดแจ้งโกง
           → สมชาย เข้า frauds (verified=true)
เจ้ามือ C: สมชาย ลงทะเบียน → เจ้ามือ C กดเช็คประวัติ
           → เจอ: "พบ 1 รายการ — ถูกแจ้งโดยผู้ใช้ในระบบ (เบอร์ตรง)"
           → เจ้ามือ C ตัดสินใจเอง: ปล่อยกู้ / ไม่ปล่อย

*** เจ้ามือ A ไม่เห็นข้อมูลของเจ้ามือ B/C ***
*** เจ้ามือ A กดเช็คประวัติ → เจอว่า สมชาย ถูกแจ้งโกง (ไม่รู้ว่าเจ้ามือ B แจ้ง) ***
```

### กด "แจ้งโกง"
```
POST /lender/debtors/:id/flag
Body: { reason: "ไม่จ่ายเงิน 3 งวด", amount: 15000, detail: "..." }

Backend:
1. ดึง debtor data (ชื่อ, เบอร์, บัญชี, บัตร ปชช)
2. เช็ค fraud ซ้ำ (เบอร์/บัญชี/บัตร ปชช)
   → ซ้ำ: ผูก fraud เดิม + reportCount++
   → ไม่ซ้ำ: สร้าง fraud ใหม่
3. สร้าง fraud_report พร้อม refCode
4. *** verified = true ทันที *** (ไม่ต้อง admin approve)
5. อัปเดต debtor: status=flagged, fraud_id=xxx
6. Return: refCode + fraud_id
```

### กด "ปลดโกง"
```
POST /lender/debtors/:id/clear
Body: { note: "ชดใช้ครบแล้ว" }

Backend:
1. อัปเดต debtor: status=cleared, cleared_at, cleared_note
2. fraud → verified = false (ไม่แสดงในผลค้นหาอีก)
3. Return: success
```

## Security & Rules

| กฎ | รายละเอียด |
|----|-----------|
| เจ้าของเงินต้อง login | จัดการลูกหนี้ต้อง JWT |
| 1 user = 1 lender profile | UNIQUE constraint on user_id |
| ลูกหนี้ไม่ต้อง login | ลงทะเบียนผ่าน invite link ได้เลย |
| ลูกหนี้ไม่รู้ว่าถูกเช็ค | หน้าลงทะเบียนไม่มีคำว่า "โกง/ตรวจสอบ" |
| สมัครซ้ำได้ | ลูกหนี้คนเดียวกันสมัครหลายเจ้ามือ → แยก record |
| เจ้าของเห็นแค่ลูกหนี้ตัวเอง | query by lender_id เสมอ |
| ผลเช็คข้ามได้ | เช็คประวัติเห็นข้อมูลจากทุก source (ไม่เปิดเผยว่าใครแจ้ง) |
| แจ้งโกง = verified ทันที | เข้า frauds + ค้นหาได้เลย |
| ปลดโกง = verified false | ออกจากผลค้นหา |
| Rate limit | ลงทะเบียน 10/วัน per IP |
| Upload | R2 `lender/{lender_id}/{debtor_id}/` |
