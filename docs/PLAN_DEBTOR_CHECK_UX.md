# Plan — ปรับ UX ตรวจสอบประวัติสมาชิก

> สถานะ: DRAFT — รอ user approve

---

## Flow ใหม่

```
สมาชิกลงทะเบียน
  → เจ้าของเห็นรายชื่อ (สถานะ "รอตรวจสอบ")
  → เจ้าของกด card → เปิด Drawer
  → เห็นข้อมูลสมาชิก + ปุ่ม "ตรวจสอบประวัติ" (ไม่เห็นปุ่มแจ้ง)
  → กดตรวจสอบ → animation ใน drawer (3 steps + progress)
  → เสร็จ → แสดงผล "ตรวจพบ X" หรือ "ไม่พบข้อมูล"
  → ถึงจะเห็นปุ่ม "แจ้งเตือน" (ถ้าต้องการ)
```

---

## Status Filter Pills

```
[ทั้งหมด 10] [รอตรวจสอบ 3] [ปกติ 5] [ถูกแจ้ง 1] [ปลดแล้ว 1]
```

| Filter | เงื่อนไข | หมายถึง |
|--------|---------|---------|
| **ทั้งหมด** | ไม่ filter | ทุกคน |
| **รอตรวจสอบ** | `checked=false` (checkedAt IS NULL) | ยังไม่เคยตรวจ |
| **ปกติ** | `status=active` + เช็คแล้ว | ตรวจแล้ว ยังปกติ |
| **ถูกแจ้ง** | `status=flagged` | ถูกแจ้งเตือนแล้ว |
| **ปลดแล้ว** | `status=cleared` | ปลดแจ้งเตือนแล้ว |

## Drawer — กฎการแสดงผล

| สถานะ | สิ่งที่เห็นใน Drawer |
|-------|---------------------|
| **ยังไม่เช็ค** | ข้อมูลบุคคล + ปุ่ม "ตรวจสอบประวัติ" (ไม่เห็นปุ่มแจ้ง) |
| **กำลังเช็ค** | Animation steps ใน drawer (minimal scan) |
| **เช็คแล้ว ไม่พบ** | "ไม่พบข้อมูล" + ปุ่ม "แจ้งเตือน" |
| **เช็คแล้ว ตรวจพบ** | "ตรวจพบ X รายการ" + รายละเอียด + ปุ่ม "แจ้งเตือน" |
| **ถูกแจ้งแล้ว (flagged)** | ข้อมูลการแจ้ง + ปุ่ม "ปลดแจ้งเตือน" |

## Animation ตอนตรวจสอบ (ใน Drawer, minimal)

```
┌─────── Drawer ───────┐
│                       │
│  ข้อมูลบุคคล (จางลง)   │
│                       │
│  ┌─────────────────┐  │
│  │  🧠 กำลังตรวจสอบ  │  │
│  │  ━━━━━━━░░░ 60%  │  │
│  │                  │  │
│  │  ✓ ค้นหาในระบบ    │  │
│  │  ● เช็คฐานข้อมูล  │  │
│  │  ○ สรุปผล        │  │
│  │                  │  │
│  │  0.8s            │  │
│  └─────────────────┘  │
│                       │
└───────────────────────┘
```

- 3 steps: ค้นหาในระบบ → เช็คฐานข้อมูล → สรุปผล
- Progress bar สีเขียว (accent)
- Timer จับเวลา
- เสร็จ → fade เป็นผลลัพธ์
- อยู่ใน drawer ไม่ต้องเปิด modal ใหม่

## ไฟล์ที่แก้

### Backend (2 ไฟล์)
- `fraud-api/infrastructure/postgres/lender_repository_impl.go` — เพิ่ม `checked` filter (WHERE checked_at IS NULL / IS NOT NULL)
- `fraud-api/interfaces/api/handlers/lender_handler.go` — parse `checked` query param (ถ้า handler ส่ง param ตรง ไม่ต้องแก้ service/repo interface ก็ได้ — เช็คว่า ListDebtors รับ params อะไร)

> หมายเหตุ: ต้องเช็คว่า ListDebtors API ปัจจุบันรับ params อะไรบ้าง อาจต้องแก้ repo interface ถ้ายังไม่รองรับ

### Frontend (2 ไฟล์)
- `fraud-web/src/app/lender/debtors/page.tsx` — เปลี่ยนคำ + เพิ่ม "รอตรวจสอบ" filter + ลบปุ่มเช็คจาก card
- `fraud-web/src/app/lender/debtors/DebtorDetailDrawer.tsx` — เปลี่ยนคำ + scan animation + ซ่อนปุ่มแจ้งถ้ายังไม่เช็ค
